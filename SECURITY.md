# Security Policy

## Reporting a Vulnerability

Use GitHub private vulnerability reporting for security issues.

- Preferred: GitHub Security Advisories for this repository.
- Fallback: if private reporting is unavailable, open a normal issue only for non-sensitive reports.

Do not post exploit details publicly.

Include:

- A clear description of the issue
- Steps to reproduce
- Affected version(s)
- Any proof-of-concept details needed to verify

## Response Expectations

- We will acknowledge receipt as soon as possible.
- We will validate impact and scope.
- We will coordinate a fix and disclosure timeline.

## Scope

This policy covers:

- CLI command execution and output handling
- Capture storage and redaction behavior
- Dependency vulnerabilities
- Packaging and release pipeline artifacts
